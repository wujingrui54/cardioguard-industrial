## 연구 하이라이트(Highlights)

- 한·중 이중 언어 정적 웹 플랫폼이 산업 현장 심정지에 대해 "감지 → 예측 → 대응 → 교육 → 데이터"의 전 과정을 폐쇄 루프로 연결한다.
- 모든 기능 모듈이 AHA 2025 ECC, ERC 2021, KACPR 2025, ISO 45001:2018에 매핑되어 심사자에게 추적 가능한 표준 출처를 제공한다.
- 1만 명 근로자 규모의 시나리오 시뮬레이션 결과, 병원 외 심정지(OHCA) 퇴원 생존율이 약 10%에서 ≥ 25%로, 1차 대응자 도달 시간 중위값이 8분에서 ≤ 3분으로 개선될 것으로 예측된다.
- 원클릭 한국어 PDF 출력과 약 430개 키의 이중 언어 국제화 계층이, 경쟁 원형들에 흔한 한·중 국경 간 도입 장벽을 제거한다.
- 산출물은 완전한 오픈소스이며 어떠한 정적 호스트에서도 바이트 단위로 재현 가능하다.

## 초록

**배경.** 병원 외 심정지(OHCA)는 산업 작업장에서 치명성이 가장 높은 급성 사건 중 하나로, 전 세계 퇴원 생존율은 10% 미만이다. 발생 후 최초 4분 이내의 목격자 심폐소생술(CPR)과 자동 심장충격기(AED) 사용은 생존을 좌우하는 가장 주요한 가변적 결정 요인이다. 그러나 산업단지는 AED 분포 희소, 낮은 CPR 인증률, 단편화된 디스패치 체계를 겪으며, 한·중 합작 단지에서는 생산 등급의 이중 언어 인터페이스 부재가 더해진다.

**목적.** *CardioGuard Industrial(工心守护 / 산업심장수호)* —— 산업 심정지 응급의 폐쇄 루프 대응을 공학적으로 구현하고 경진대회 제출이 가능한 산출물로 제공하는, 재현 가능한 한·중 이중 언어 디지털 헬스 플랫폼 —— 을 설계·구현·평가한다.

**방법.** 우리는 실시간 생체신호 모니터링, 인공지능(AI) 위험도 예측 사양, AED 사물인터넷(IoT) 네트워크, 음성 안내 CPR 대응, 디지털 교육, 안전 데이터 콕핏을 통합한, 의존성 없는(zero-dependency) 접근성 준수(WCAG 2.1 AA) 정적 웹 원형을 개발하였다. 각 모듈은 AHA 2025 ECC, ERC 2021, KACPR 2025, ISO 45001:2018에 매핑하였다. 본 산출물은 최소 기능 제품(MVP)이므로, 기대 효과는 일차 데이터 수집이 아닌 문헌 삼각검증과 1만 명 근로자 규모 중형 산업단지에 대한 시나리오 시뮬레이션으로 평가하였다.

**결과.** 본 원형은 한·중 네이티브 전환(약 430개 국제화 키), 원클릭 한국어 PDF 출력, 4개의 인터랙티브 대시보드를 제공한다. 발표된 기준선과 비교했을 때 시나리오 시뮬레이션은 다음을 예측하였다: OHCA 퇴원 생존율 약 10% → ≥ 25%(+15%p), 쓰러짐 → CPR 시간 중위값 약 8분 → ≤ 3분(−62%), 4분 내 AED 도달성 38% → 91%(+53%p), CPR 인증률 12% → ≥ 60%. 1만 명 단지 기준 추정 투자수익률(ROI) > 5.

**결론.** CardioGuard Industrial은 표준에 기반하고 이중 언어를 지원하는 오픈소스 원형으로서, 비용 없이 재현 가능하며 한·중 합작 단지의 시범 도입으로 이행할 준비가 되어 있다. 본 연구는 *산업 안전* 과제에 대한 실행 가능한 디지털 헬스 응답이자, 동료심사를 통한 확장의 템플릿을 제시한다.

**키워드:** 디지털 헬스; 산업 안전; 병원 외 심정지; 자동 심장충격기; 심폐소생술; 사물인터넷; 임상 의사결정 지원; 이중 언어 사용자 인터페이스; 한·중 협력; 공중보건 정보학

## 1. 서론

### 1.1 산업 현장 OHCA의 공중보건 부담

미국심장협회(AHA) 2025 *Heart Disease and Stroke Statistics* 갱신 보고서에 따르면, 산업화 경제권의 OHCA 연간 발생률은 인구 10만 명당 50–110건이며, 대부분의 대규모 등록 연구에서 퇴원 생존율은 8.0–10.4%이다 [1]. 산업 근로자는 긴 교대 주기, 고온·소음 노출, 분진 흡입, 만성 스트레스 등 잘 알려진 부정맥 조절 요인으로 인해 위험도가 더 높다 [2,3]. 산업단지 내에서는 네 가지 기전이 효과적 대응을 지속적으로 지연시킨다:

1. **AED 희소성과 열화.** 장비는 설치되었으나 거의 유지보수되지 않는다. 감사 대상이 된 아시아 배치에서 공공 AED의 약 22%가 필요 시점에 작동하지 않았다 [4].
2. **낮은 CPR 인증률.** 한·중 제조업 설문에서 비의료 직원의 자가보고 인증률은 8–14%에 불과하다 [5].
3. **디스패치 단편화.** 사내 의무실, 훈련된 자원봉사자, 119/120 응급센터가 서로 단절된 정보 채널에서 운영된다.
4. **언어 마찰.** 한·중 합작 단지(예: 산둥, 평택)에는 생산 등급의 이중 언어 안전 인터페이스가 없다.

### 1.2 디지털 헬스 × 산업 안전의 기회

웨어러블 생체감지, 엣지 AI, IoT 기반 제세동의 융합은 실행 가능한 설계 공간을 만든다. 세계보건기구 *Global Strategy on Digital Health 2020–2025* [6], 국제노동기구 *Vision Zero Fund* 프로그램 [7], 대한민국 제5차 *산업재해 예방 종합계획*(2023) [8]은 모두 작업장 응급 역량의 디지털화를 우선순위로 둔다.

### 1.3 기존 접근과 공백

기존 시스템은 문제의 단편만 다룰 뿐 전체 사슬을 다루지 못한다(표 1). PulsePoint(미국) [12], AED Map(한국) [13] 같은 AED 지리정보 서비스는 공공 공간을 대상으로 하며 기업 환경-보건-안전(EHS) 시스템과 통합되지 않는다. Apple Watch [14] 같은 소비자용 웨어러블 심전도는 기업급 위험 분류가 없다. CPR 온라인 학습(AHA HeartCode; KACPR e-Learning [11])은 실시간 디스패치와 단절되어 있다. 표준 119/120 병원 전 절차는 병렬 자원봉사자 디스패치를 제공하지 않는다. 이러한 요소들을 네이티브 이중 언어 제공과 함께 수직적 폐쇄 루프로 통합한 시스템은 이전에 없었다.

### 1.4 목표와 기여

본 연구의 기여는 다음과 같다:

- 디지털 헬스 경진대회 *산업 안전* 주제를 겨냥한, 완전 이중 언어(한 ↔ 중)·의존성 없는·접근성 준수 웹 원형;
- AHA 2025 [9], ERC 2021 [10], KACPR 2025 [11]에 매핑된, 모니터링·예측·디스패치·교육·분석을 아우르는 통합 사양;
- 기대 생존율, 대응 시간, ROI, 지속가능발전목표(SDG) 영향을 정량화하는 문헌 삼각검증 및 시나리오 시뮬레이션 기반 평가;
- 심사자와 후속 연구자가 수 분 내에 복제·수정·재배포할 수 있는 재현 가능한 오픈소스 공개(MIT).

## 2. 방법

### 2.1 설계 원칙과 요구사항

플랫폼은 다섯 가지 원칙으로 사양화하였다: (i) EHS 관리자, 현장 근로자/자원봉사자, 규제자라는 세 역할 페르소나를 위한 **사용자 중심** 설계; (ii) 모든 지표가 정량화·추적·내보내기 가능한 **데이터 기반** 운영; (iii) AHA 2025 ECC [9], ERC 2021 [10], KACPR 2025 [11], ISO 45001:2018 [15]과의 **표준 정렬**; (iv) 프런트엔드 무저장, 클라이언트 측 PDF 생성, HTTPS 강제를 갖춘 **설계 단계 프라이버시(privacy by design)**; (v) 오프라인 동작 가능한 데모 코어를 갖춘 정적 프런트엔드를 통한 **경량 접근성**.

### 2.2 참조 아키텍처

플랫폼은 다섯 계층으로 사양화된다(그림 1): 웨어러블 계층(단일 유도 심전도 [ECG], SpO₂, 피부 온도); 엣지 계층(게이트웨이와 실시간 부정맥 식별); 서비스 계층(Shapley 가산 설명 [SHAP] 해석성 [17]을 갖춘 그래디언트 부스팅 위험 모델과 디스패치 최적화); 응용 계층(본 MVP로 구현된 이중 언어 웹 클라이언트); 표준 브리지(AHA/ERC/KACPR/ISO/HL7 FHIR). 경진대회 원형은 응용 계층을 구현하며, 시뮬레이션 데이터로 전체 상류 가치 사슬을 보여준다.

```
┌─────────────────────────────────────────────────────────┐
│ ① 웨어러블 계층:  ECG 패치 · SpO₂ · 피부 온도            │
├─────────────────────────────────────────────────────────┤
│ ② 엣지 계층:      게이트웨이 · 실시간 부정맥 식별        │
├─────────────────────────────────────────────────────────┤
│ ③ 서비스 계층:    그래디언트 부스팅 위험 + SHAP          │
├─────────────────────────────────────────────────────────┤
│ ④ 응용 계층(본 MVP): 이중 언어 웹 · PDF                  │
├─────────────────────────────────────────────────────────┤
│ ⑤ 표준 브리지:    AHA / ERC / KACPR / ISO / HL7 FHIR     │
└─────────────────────────────────────────────────────────┘
```

**그림 1.** 계층형 참조 아키텍처. 본 논문이 보고하는 최소 기능 원형은 ④ 계층을 구현하며, ①–③ 계층과 ⑤ 계층은 후속 시범 도입을 위해 사양화되었다.

### 2.3 프런트엔드 구현

클라이언트는 재현성을 극대화하고 페이지 용량을 최소화하기 위해 빌드 없는·프레임워크 없는 스택으로 구축하였다(표 2). 데이터 시각화는 Chart.js 4.4를, 범-CJK 타이포그래피는 Noto Sans SC / Noto Sans KR / Inter를 사용한다.

### 2.4 국제화

경량 `data-i18n` 속성 매핑(로케일당 약 430개 키)이 간체 중국어와 한국어 간 전체 사용자 인터페이스 번역을 구동한다. 사전은 `js/i18n.js`에 함께 배치되며, 언어 선호는 `localStorage`에 영구 저장되어 재방문 시 다시 적용되고, URL 매개변수(`?lang=zh|ko`)로 특정 로케일에 딥링크할 수 있다. 전환 시 차트 축 레이블이 다시 바인딩되어 주석이 현재 로케일과 일치한다.

### 2.5 이중 언어 PDF 생성

일부 Chromium 빌드에서 관찰된 html2canvas 공백 출력 장애를 제거하기 위해, 기본 다운로드 경로는 Python `fpdf2` 라이브러리와 Windows 내장 Malgun Gothic 글꼴로 빌드한 사전 생성 정적 PDF(`assets/CardioGuard_Industrial_KR.pdf`, 122 kB)를 제공하여, 어떤 브라우저 캔버스와도 무관하게 충실한 한국어 렌더링을 보장한다. 정적 파일에 도달할 수 없으면, 버튼은 DOM 요소 참조 대신 HTML 문자열 입력으로 호출되는 클라이언트 측 `html2pdf.js`로 폴백하여 화면 밖 위치 지정 버그를 회피한다. 내보낸 PDF는 경진대회 평가 기준이 요구하는 네 개 섹션을 포함한다: (1) 웹사이트 설계 및 개발 방법; (2) 주요 기능; (3) 기대 효과 및 응용 가치; (4) 보충 설명.

### 2.6 평가 방법

최소 기능 원형으로서, 플랫폼은 일차 데이터 수집이 아닌 **문헌 삼각검증** 과 **시나리오 시뮬레이션** 으로 평가하였다. 기준선은 AHA 2025 [9], ERC 2021 [10], KACPR 2025 [11]에서 도출하였고, 도입 시나리오는 1만 명 근로자 규모의 중형 산업단지로 가정하였다. Chart.js 대시보드는 핵심 운영 및 역학 지표에 대한 도입 전·후 차이를 보여준다.

## 3. 결과

### 3.1 기능 모듈

원형은 전체 응급 루프를 아우르는 여섯 개 기능 모듈을 구현하며, 각각은 임상 지침 조항으로 추적 가능하다(표 3).

### 3.2 기대 임상 및 운영 효과

시나리오 시뮬레이션은 모델링된 모든 지표에서 상당한 개선을 예측하였다(표 4). 가장 큰 절대 이득은 4분 내 AED 도달성(+53%p)과 OHCA 퇴원 생존율(+15%p)에서 나타났으며, 1만 명 단지에서 고비용 급성 사건의 회피로 인해 모델링된 ROI > 5였다.

### 3.3 지속가능발전목표와의 정합성

플랫폼은 SDG 3.4(비전염성 질환 조기 사망 감소), SDG 8(양질의 일자리와 산업 근로자 생명 보호), SDG 17(국경 간 파트너십, 여기서는 네이티브 이중 언어 제공으로 촉진)을 진전시킨다.

## 4. 고찰

### 4.1 주요 발견

CardioGuard Industrial은 산업 심정지의 전 과정—모니터링, 예측, AED 디스패치, 음성 안내 대응, 인증 교육, 분석—이 표준에 기반하고 비용 없이 재현 가능한 단일 이중 언어 산출물로 통합될 수 있음을 보여준다. 예측 효과 크기(표 4)는 조기 목격자 CPR과 조기 제세동의 확립된 생존 이익과 일치하여 [1,9,10], 시뮬레이션의 표면 타당도를 뒷받침한다.

### 4.2 기존 연구와의 비교

공공 공간 AED 지도 [12,13], 소비자 웨어러블 [14], 독립형 온라인 학습 [11]과 달리, 본 설계는 산업단지 시나리오 내에서 사슬을 *수직적으로* 폐쇄하고 네이티브 이중 언어 제공을 더한다(표 1). 우리가 아는 한, 이 조합—산업 심정지 폐쇄 루프 + 생산 등급 한·중 동등성 + 보장된 한국어 PDF—은 이전에 보고된 바 없다.

### 4.3 응용과 한·중 산업 협력

직접 수혜자에는 산업 운영자(낮은 사건 비용, 향상된 환경-사회-지배구조 등급), 근로자(접근 가능한 구명 대응), 공중보건 규제자(실시간 단지급 역학), 보험사(직업성 심장 위험의 정밀 보험계리 가격 책정)가 포함된다. 후보 국경 간 도입지로는 산둥 옌타이 한국 자본 산업단지, 경기 평택 중국 자본 제조 구역, 웨이하이 한·중 자유무역 시범구가 있다.

### 4.4 규제 경로

제품화를 위해, 플랫폼은 중국에서 국가약품감독관리국 디지털 헬스 프레임워크에 따라 II 등급 의료기기 소프트웨어(SaMD)로 분류될 수 있고, 대한민국에서는 식품의약품안전처(MFDS) 디지털 헬스 소프트웨어 시판 전 신고(2024) [18]를 거치며, 국제적으로는 IEC 62304(의료 소프트웨어 수명주기)와 ISO 13485(품질 경영)에 정렬되어야 한다.

### 4.5 한계

첫째, 데모 데이터는 공개 통계로부터 시뮬레이션한 것으로, 검증된 산업 코호트는 아직 수집되지 않아 예측 효과 크기는 실증적이라기보다 예시적이다. 둘째, AI 위험 모듈은 본 MVP에서 사양화되었을 뿐 배포·벤치마크되지 않았다. 셋째, 프라이버시와 보안은 프런트엔드 모범사례 수준에서 구현되었으며, 전 주기 감사(ISO/IEC 27001)는 미완이다. 넷째, 언어 커버리지는 한·중에 한정되어 동남아시아 노동시장은 아직 다루지 못한다.

### 4.6 향후 과제

로드맵은 한·중 합작 기업 2개 공장 시범(2026 Q3)과 500명 전향적 코호트(2026 Q4)에서, MFDS 시판 전 신고(2027 Q1), 동료심사 발표 및 10개 단지 확장(2027 Q2), 동남아시아 언어 확장(2027 Q4)으로 진행된다. 모든 전향적 코호트는 이중 기관생명윤리위원회 감독과 헬싱키 선언 원칙 하에 수행될 것이다.

## 5. 결론

CardioGuard Industrial은 디지털 헬스 경진대회 *산업 안전* 요청에 응답하는, 재현 가능하고 표준에 정렬된 이중 언어 원형을 제공한다. 실시간 모니터링, AI 위험도 예측, AED IoT 디스패치, 음성 안내 CPR 대응, 인증 교육, 안전 분석을 단일 폐쇄 루프로 통합하고—배포된 웹 클라이언트, MIT 라이선스 소스 코드, 원클릭 한국어 PDF로 산출물을 패키징함으로써—본 산출물은 임상 지침 충실도, 이중 언어 접근성, 오픈소스 재현성을 결합한다. 이러한 특성은 한·중 합작 단지의 시범 도입으로의 이행과 후속 동료심사 발표에 매우 적합하다.

## 선언

### 윤리 승인 및 참여 동의
해당 없음. 본 최소 기능 원형은 인간 피험자, 인간 데이터, 동물 대상을 포함하지 않으며 기관생명윤리위원회 승인이 필요하지 않았다. 후속 전향적 코호트(§4.6)는 시안의학고등전문학교와 경운대학교의 이중 심사를 받고 헬싱키 선언 원칙 하에 수행될 것이다.

### 출판 동의
해당 없음.

### 데이터 및 자료 가용성
본 연구의 소스 코드, 시뮬레이션 대시보드 데이터, 생성된 한국어 PDF는 프로젝트 저장소에서 MIT 라이선스(코드)와 CC BY 4.0(논문 본문)으로 공개된다. 인간 피험자 데이터는 포함되지 않는다. 산출물은 저장소 루트에서 `python -m http.server`로 로컬에서 재현할 수 있다.

### 이해 상충
저자는 이해 상충이 없음을 선언한다.

### 연구비
본 연구는 외부 연구비를 받지 않았다. *(해당 시 투고 시점에 갱신.)*

### 저자 기여
J.W.가 연구를 구상하고 플랫폼을 설계·구현하였으며 평가를 수행하고 원고를 작성하였다. 저자는 최종 원고를 읽고 승인하였다.

### 감사의 글
저자는 비공식 토론에 응해 준 시안의학고등전문학교와 경운대학교 동료들에게 감사한다.

### 인공지능 사용(ICMJE/COPE 정렬 공개)
생성형 AI는 **오직 저자가 작성한 산문의 언어 다듬기와 상용구 코드 스캐폴딩**(예: CSS 그래디언트, ARIA 스니펫, 빌드 구성)에만 사용되었다. 저자는 모든 코드 한 줄, 모든 임상 인용, 모든 정량 예측, 본 원고의 모든 단락을 검토하고 전적인 책임을 진다. **본 연구 과정에서 어떠한 환자·피험자·임상기록 데이터도 AI 시스템에 제공되지 않았다.**
